Cyber Threats

Here are 10 types of cyber threats in no particular order:

  • Malware

    Malware (short for "malicious software") is a software/program that is typically used to damage, disrupt or gain unauthorized access to a computer or computer system.

  • Phishing

    Phishing is a type of social engineering where somebody with ill intentions sends a fake message (usually an email) to trick their victim into sharing revealing information with them, such as accounts and passwords.

  • Ransomware

    Ransomware (short for "ransom software") is a software/program that locks the victims computer in exchange usually for money, or potentially whatever the attacker wants.

  • Outdated Hardware

    As hardware becomes obsolete, out of date devices will not be able to support updates to protect the system from potential cyber attacks in the future.

  • Denial-of-service Attack

    A denial-of-service (commonly shortened to "DDos") attack is when an attacker sends mulitple requests to the targeted website with the intention of overloading the websites capability to handle all the requests, resulting in the website functioning incorrectly.

  • Cloud Vulnerabilities

    Cloud vulnerabilities are when the storage cloud an individual or company might use is vulnerable, in which case attackers may take advantage of this and try gain unauthorized access to the cloud.

  • Social Engineering

    Social engineering is a tactic attackers use to deceive and trick victims into sharing personal information, examples of which include requesting urgent help or offering large sums of money which follow into requesting the victims information.

  • SQL Injection

    SQL (short for "Structured Query Language") injection is a code injection technique used to attack data-driven applications, in which malicious SQL statements are inserted into an entry field for execution.

  • Zero-day Exploit

    A zero-day exploit is the method hackers use to attack systems with a previously unidentified vulnerability. Zero-day is a term that means the developer has just learned of the flaw, and thus they have "zero days" to fix it.

  • DNS Attack

    A DNS attack is an exploit where the attacker takes advantages of vulnerabilities in the domain name system (DNS).
